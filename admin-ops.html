Milestones & alerts

These definitions drive the traffic lights (Green / Amber / Red) every operator sees on the worklist and shipment drawer — the Alert timing decides when a pending milestone turns Amber/Red. baseline = expected window from 3-yr lane history; fixed = a hard offset from an anchor date field (e.g. 3 days before onboard1); none = no time alert (Green until missed outright). Changes apply to each shipment at its next evaluation run (Air ~2 h, Sea 3×/day, or a manual seed-alerts.ps1 re-run) — already-painted lights update then, for all users.

Generate documents

Each row is a document an operator can generate from the ERP on a shipment (the worklist drawer's Generate document box). Pick the module, then the Document type code and the related House type code — both must match the ERP codes exactly (e.g. AWB / NEUTRAL L 23); they are sent verbatim to /document/generate. One document type can have several house types (add a row per pair). Tick Master bill for a master-level document (it keys the call on the master bill); tick Invoice if the document needs an invoice number entered. Changes apply immediately — no restart.

ERP API identity

Codes sent to the Swivel ERP on every Save data to ERP / document call. Party group code is the company/customer code (e.g. DEV) — required by the ERP. Forwarder code is the office owncode the data belongs to (where it goes, e.g. S0001=HKG); this is the fallback — each station's real owncode is resolved automatically from the ERP when available. Changes apply immediately — no restart.

ERP connection

The Swivel ERP HTTP API the app pushes to. Stored in the database (overrides the server config; no file edit, no restart). Real ERP calls happen only when a Base URL and a token are set and "use mock" is off — otherwise calls are mocked (nothing is sent). Status: —.
Tokens: {station} {m}(A/S) {mode}(AIR/SEA) {yymmdd} {yy} {seq}/{seq5}. Must include a {seq} token. E.g. {station}{m}{seq5} → HKGA00001. Blank = default.
Prefix for the customer document-review links → <url>/bl-review/<token>. Set to the internet-facing HTTPS host so customers can open it. Blank = config file (else http://localhost:<port>).
For IT support: process health + storage growth — no database access needed. Detailed records are in the Change log tab.
