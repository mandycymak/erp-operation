Milestones & alerts

These definitions drive the traffic lights (Green / Amber / Red) every operator sees on the worklist and shipment drawer — the Alert timing decides when a pending milestone turns Amber/Red. baseline = expected window from 3-yr lane history; fixed = a hard offset from an anchor date field (e.g. 3 days before onboard1); none = no time alert (Green until missed outright). Changes apply to each shipment at its next evaluation run (Air ~2 h, Sea 3×/day, or a manual seed-alerts.ps1 re-run) — already-painted lights update then, for all users.

ERP API identity

Codes sent to the Swivel ERP on every Save data to ERP / document call. Party group code is the company/customer code (e.g. DEV) — required by the ERP. Forwarder code is the office owncode the data belongs to (where it goes, e.g. S0001=HKG); this is the fallback — each station's real owncode is resolved automatically from the ERP when available. The bearer token is not here (it stays in the server config). Changes apply immediately — no restart.
